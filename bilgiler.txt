GİT VERSİYON KONTROL SİSTEMİ

Git, açık kaynak kodlu ve dağıtık bir versiyon kontrol sistemidir. Yazılım projelerinde değişikliklerin izlenmesi ve yönetilmesi için kullanılır.

Linus Torvalds tarafından 2005 yılında linux çekirdek geliştirme sürecini desteklemek amacıyla geliştirilmiştir. 

Benzerleri: Subversion (svn), Mercurial, Perforce, Bazaar, tfs

GİTHUB

Git versiyon kontrol sistemini kullanarak projelerimizi barındırmamıza, yönetmemize ve paylaşmamıza olanak sağlayan bulut tabanlı bir platformdur. 2008 yılında kurulan ve 2018 yılında microsoft tarafından satın alınan github dünya çapında milyonlarca geliştirici tarafından kullanılmaktadır. 

Benzerleri: Gitlab, Bitbucket, SourceForge, Gitea, Gogs gibi.

pwd : bulunduğunuz dizin
cd  : klasörler arası geçiş 
clear   : ekranı temizler
cd 'dosyayolu'  : istenilen dizine gider.
touch   : dosya oluşturmaya yarar.
ls      : bulunduğunuz dizindeki dosyaları gösterir
git init    : konumlandığınız klasöre git versiyon kontrol sistemini kurar.
git add <dosya adı> : git tarafından kontrol edilmesini yani versiyonlanmasını isdeiğimiz dosyaları staging area'ya ekler.

git add .   : tüm dosyaları staging area'ya ekler. Yani tüm dosyalar versiyonlanmak üzere takibe alınır.
git status  : son durumu gösterir.
git commit -m "açiklama" : staging area da bulunan tüm dosyalar local repoya eklendi.

index.html içerisinde değişiklik yapıldı. 

git add index.html                  --staging
git commit -m "Header Eklendi"      --commitle

git status  : son değişiklikleri kontrol eder.
